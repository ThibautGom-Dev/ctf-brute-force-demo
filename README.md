# ctf-brute-force-demo
Educational CTF lab demonstrating the risks of using personal information in passwords through automated combinatorial attacks.

## ⚠️ Disclaimer
**This project is for educational purposes only.** It demonstrates how easily passwords based on personal information can be compromised. The author assumes no responsibility for misuse.

## Project Structure
```text
.
├── bruteforce_ctf.py       # Main cracking engine
├── hash_generator.py       # Tool to create new target hashes
├── hashes.txt              # List of target passwords (hashes)
├── index.html              # Simulation login page
├── target_profile.txt      # Information to analyze (OSINT)
└── words.txt               # User-created dictionary

```

## Setup
1. Analyze: Read `target_profile.txt` to find keyworlds.
2. Configure: Setup `words.txt` file and populate it with potential passwords (e.g., john, ...).
3. Run: Put the hash level in `hashes.txt` and execute the script: `python bruteforce_ctf.py`.
4. Verify: Once the password is found, test it on the `index.html` login page.

## Lesson
Password complexity is a myth if you rely on personal data. An automated tool can test thousands of combinations in seconds.
Defense: Use long, random passphrases generated by a Password Manager.
